# rv64

The `rv64` profile models 64-bit RISC-V supervisor mode with 4 KiB pages.

Implementation:

- `src/arch/rv64/ops.cpp`: architecture operations.
- `src/arch/rv64/boot.S`: direct entry, bootstrap stack setup, `.bss` clearing,
  and call into `kernel_main`.
- `src/arch/rv64/platform.cpp`: QEMU virt NS16550 UART platform I/O.
- `src/arch/rv64/linker.ld`: ELF layout at the QEMU virt DRAM base.

The profile provides `stvec`/PLIC/SBI metadata, `ecall` syscall metadata, `sret`
transition metadata, and guarded RISC-V inline assembly for `rdcycle`, `fence`,
SIE control, and wait/yield behavior.

## Bring-up Plan

- Interrupts: trap vector, PLIC for external interrupts, CLINT/SBI timer path.
- Memory: Sv39 first, optional Sv48 later.
- User mode: `sret` from supervisor mode to user mode.
- Syscalls: `ecall` from user mode.
- QEMU: `qemu-system-riscv64 -M virt -bios none -kernel` boots the linked ELF
  at `0x80000000` and validates the debug kernel through serial output.
