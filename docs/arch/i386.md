# i386

The `i386` profile models 32-bit x86 with 4 KiB pages, little-endian memory, and
ring 3 user mode.

Implementation: `src/arch/i386/ops.cpp`.

The profile provides IDT/PIC/APIC metadata, `int 0x80`/`sysenter` syscall
metadata, `iret`/TSS user-transition metadata, and guarded x86 inline assembly
for cycle counters, memory fences, interrupt control, and halt/yield behavior.

## Bring-up Plan

- Interrupts: IDT with interrupt/trap gates and legacy PIC/APIC routing.
- Memory: two-level paging, higher-half kernel mapping, user/supervisor page
  permissions.
- User mode: `iret` transition to ring 3 with TSS-managed kernel stack.
- Syscalls: `int 0x80` first, `sysenter` later for fast paths.
- QEMU: `qemu-i386` for user-mode smoke, `qemu-system-i386` for CPU/IDT tests.
